# WebSocket 错误代码参考手册

本文档列出 WebSocket 标准关闭代码及 AutoDial 项目中的自定义用法。

---

## 标准 WebSocket 关闭代码（1000-1015）

| 代码 | 名称 | 含义 | 常见场景 |
|------|------|------|---------|
| **1000** | CLOSE_NORMAL | 正常关闭 | 主动断开连接 |
| **1001** | CLOSE_GOING_AWAY | 端点离开 | 网络断开、服务器关闭 |
| **1002** | CLOSE_PROTOCOL_ERROR | 协议错误 | 数据格式错误 |
| **1003** | CLOSE_UNSUPPORTED | 不支持的数据类型 | 发送了不支持的数据 |
| **1004** | **保留（私有使用）** | **自定义错误代码** | ⚠️ AutoDial 用此表示认证失败 |
| **1005** | CLOSE_NO_STATUS | 没有收到关闭帧 | 连接异常中断 |
| **1006** | CLOSE_ABNORMAL | 连接异常关闭 | 网络断开、服务器崩溃 |
| **1007** | INVALID_PAYLOAD | 数据格式错误 | JSON 解析失败 |
| **1008** | POLICY_VIOLATION | 策略违规 | 未授权访问 |
| **1009** | MESSAGE_TOO_BIG | 消息过大 | 发送了超大消息 |
| **1010** | MANDATORY_EXTENSION | 扩展 required | 协议扩展不匹配 |
| **1011** | INTERNAL_ERROR | 服务器内部错误 | 服务器崩溃 |
| **1012** | SERVICE_RESTART | 服务重启 | 服务器重启中 |
| **1013** | TRY_AGAIN_LATER | 稍后重试 | 服务器过载 |
| **1014** | BAD_GATEWAY | 网关错误 | 代理/网关问题 |
| **1015** | TLS_HANDSHAKE_FAILED | TLS 握手失败 | HTTPS/WSS 证书错误 |

## AutoDial 项目中的自定义用法

### Android 端（`DialService.kt`）

```kotlin
// 正常关闭
ws.close(1000, null)

// 认证失败（配对码错误）
ws.close(1000, "auth_fail")
```

### PC 端（`main.js`）

```javascript
// 正常关闭
cloudWs.close(1000, '正常关闭');

// 错误场景（使用标准代码）
cloudWs.close(1008, '未授权');  // 策略违规
cloudWs.close(1011, '服务器内部错误');  // 内部错误
```

⚠️ **注意**：代码中可能用 `1004` 表示自定义错误（非标准用法），建议改为标准代码 `1008`（策略违规）。

## 常见错误场景和解决方法

### 错误：1001（端点离开）
**原因：**
- 网络断开
- 服务器关闭
- 手机/电脑端主动退出

**解决方法：**
- 检查网络连接
- 检查服务器是否运行
- 自动重连

---

### 错误：1004（自定义 - 认证失败）
**原因：**
- 配对码（PIN）错误
- 服务器拒绝了连接

**解决方法：**
- 检查手机和电脑端的配对码是否一致
- 重新生成配对码
- 检查服务器配置

---

### 错误：1006（连接异常关闭）
**原因：**
- 网络断开
- 服务器崩溃
- 防火墙拦截

**解决方法：**
- 检查网络连接
- 检查服务器是否运行
- 检查防火墙设置
- 自动重连

---

### 错误：1005（无状态关闭）
**原因：**
- 连接被强制中断
- 对方没有发送关闭帧

**解决方法：**
- 自动重连
- 检查网络稳定性

---

### 错误：1008（策略违规 - 未授权）
**原因：**
- 未授权访问
- 配对码错误
- IP 被拉黑

**解决方法：**
- 检查配对码
- 检查服务器授权配置

---

### 错误：1011（服务器内部错误）
**原因：**
- 服务器崩溃
- 服务器代码错误

**解决方法：**
- 重启服务器
- 查看服务器日志

---

### 错误：1015（TLS 握手失败）
**原因：**
- WSS 证书错误
- 证书过期
- 证书不匹配

**解决方法：**
- 检查证书是否有效
- 如果使用自签名证书，客户端需要信任

## Node.js 网络错误代码（补充）

除了 WebSocket 关闭代码，PC 端还可能遇到 Node.js 网络错误：

| 错误代码 | 含义 | 解决方法 |
|---------|------|---------|
| `ECONNREFUSED` | 连接被拒绝 | 服务器未运行，检查地址和端口 |
| `ETIMEDOUT` | 连接超时 | 检查网络，服务器可能挂了 |
| `EHOSTUNREACH` | 主机不可达 | 检查 IP 地址是否正确 |
| `ENETUNREACH` | 网络不可达 | 检查网络连接 |
| `ECONNRESET` | 连接被重置 | 服务器强制断开，检查防火墙 |

## Android 端网络错误

Android 端使用 OkHttp，常见异常：

| 异常 | 含义 | 解决方法 |
|------|------|---------|
| `java.net.ConnectException` | 连接失败 | 检查服务器地址和端口 |
| `java.net.SocketTimeoutException` | 连接超时 | 检查网络，增加超时时间 |
| `java.net.UnknownHostException` | 域名解析失败 | 检查域名是否正确 |
| `javax.net.ssl.SSLException` | SSL 错误 | 检查证书，或使用 HTTP 替代 HTTPS |

## 建议改进

1. **统一错误代码**：建议 PC 端和 Android 端使用相同的错误代码
2. **友好提示**：将错误代码转换为用户友好的中文提示
3. **日志记录**：在日志中记录完整的错误信息，方便调试

## 示例：友好错误提示

```kotlin
// Android 端示例
fun getErrorMessage(code: Int): String {
    return when (code) {
        1000 -> "连接已关闭"
        1001 -> "网络断开，正在重连..."
        1004 -> "配对码错误，请检查配置"
        1006 -> "连接异常，正在重连..."
        1008 -> "未授权，请检查配对码"
        1011 -> "服务器错误，请稍后重试"
        else -> "未知错误（代码：$code）"
    }
}
```

```javascript
// PC 端示例
function getErrorMessage(code) {
    const messages = {
        1000: '连接已关闭',
        1001: '网络断开，正在重连...',
        1004: '配对码错误，请检查配置',
        1006: '连接异常，正在重连...',
        1008: '未授权，请检查配对码',
        1011: '服务器错误，请稍后重试'
    };
    return messages[code] || `未知错误（代码：${code}）`;
}
```
