## 一、项目概览

**跨屏拨号**（AutoDial）是一个电销一键拨号系统，由 4 个端组成：

| 端 | 技术栈 | 通信协议 | 功能 |
|---|--------|---------|------|
| PC 端 | Electron + Node.js | WebSocket / HTTP | 拨号控制台、号码管理、短信模板 |
| Android 端 | Kotlin + OkHttp | WebSocket / HTTP | 执行拨号/挂断、通话记录、统计 |
| Chrome 插件端 | JavaScript | HTTP → PC 端 | 网页浮窗拨号、CRM 集成 |
| 云服务器端 | Python + websockets | WebSocket 中转 | PIN 分组转发、跨网络连接 |

### 通信架构
```
[PC 端] ←WebSocket→ [云服务器] ←WebSocket→ [Android 端]
   ↕ LAN直连（优先）                              ↕
[Chrome 插件] ——HTTP /open——→ [PC 端]           ↕
                                             执行拨号
```

**双通道策略**：局域网 WiFi 直连优先，断开后自动切换云端中转。

## 三、端口与协议

| 用途 | 端口 | 协议 |
|------|------|------|
| PC 端 ↔ 手机端（LAN 直连） | 35432 | WebSocket + HTTP |
| 云中转服务器 | 35430 | WebSocket + HTTP（/health） |

### WebSocket 消息协议

**握手**：
- PC → 服务器：`{"type":"pc_hello","pin":"6625","hostname":"xxx"}`
- 服务器 → PC：`{"type":"pc_auth_ok","pin":"6625","phoneCount":0}`
- 手机 → 服务器：`{"type":"phone_hello","pin":"6625","deviceName":"xxx"}`
- 服务器 → 手机：`{"type":"auth_ok","pin":"6625","pcCount":1}`

**业务消息**（PC→手机 / 手机→PC 双向转发）：
- `dial`：拨号指令 `{"type":"dial","pin":"xxx","number":"13800138000"}`
- `dial_result`：拨号结果 `{"type":"dial_result","number":"xxx","status":"success"}`
- `hangup`：挂断指令
- `sms`：短信指令
- `sms_result`：短信发送结果
- `ping/pong`：心跳（ping 不记日志）

### HTTP 接口（PC 端 35432）
- `GET /open` — 唤起 PC 端主窗口（插件调用）
- `GET /status` — 连接状态查询
- `GET /dial?number=xxx` — HTTP 拨号

### HTTP 接口（云服务器 35430）
- `GET /health` — 健康检查 `{"service":"AutoDial Cloud Relay","version":"1.0.0","groups":1,"connections":2,"port":35430}`

## 四、关键配置

### APK 签名
- 格式：PKCS12，alias=autodial，密码=AutoDial2026!
- GitHub Secrets：KEYSTORE_BASE64, KEYSTORE_PASSWORD, KEY_ALIAS, KEY_PASSWORD
- CI 流程解码 p12 文件到 `android/app/`

### 云中转部署
- **本地运行**：双击 `AutoDial-Cloud-Relay.exe`，默认端口 35430
- **指定端口**：`AutoDial-Cloud-Relay.exe --port 35431`
- **系统托盘**：绿色圆点=运行中，灰色圆点=已停止
- **日志**：`%APPDATA%/autodial-cloud-relay/cloud-relay.log`
- **公网访问**：DDNS `262ao85kz470.vicp.fun:35430`（需端口转发）

### Android 端云中转配置
- 开关：设置页 → 云中转连接
- 地址输入：直接输入 IP:端口（如 `1.2.3.4:35430`），代码自动补 `ws://`
- 配对码：4 位数字 PIN，PC 和手机需一致
- 连接成功后 Toast 提示 `✅ 云服务器连接成功`

## 五、构建方法

### PC 端
```bash
cd pc-app/
npm install
npm run build      # 或 node pack.js
# 输出：pc-app/output/AutoDial-win32-x64/AutoDial.exe
```

### Android 端
- **GitHub Actions**：push 到 master 分支自动构建
- **本地构建**：`cd android && ./gradlew assembleRelease`
- **APK 下载**：https://github.com/ztj555/apk/actions

### 云服务器 EXE
```bash
cd cloud-relay/python/
pip install -r requirements.txt    # 安装依赖
build.bat                          # PyInstaller 打包
# 输出：cloud-relay/python/dist/AutoDial-Cloud-Relay.exe
```

### Chrome 插件
- Chrome → 扩展程序 → 开发者模式 → 加载已解压的扩展程序 → 选择 `插件端/源码` 目录

ThemeManager 支持 16 套主题 × 7 级亮度，tag-based 递归着色。

## 八、多手机连接

- 支持多台 Android 设备同时连接同一 PC
- PC 端可切换活跃设备、添加设备备注
- 手机列表：局域网手机正常显示，云端手机标记 `isCloud=true, ip='cloud'`
- `sendToPhone()` 方法自动选择目标设备
