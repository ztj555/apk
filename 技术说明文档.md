## 一、项目概述

**AutoDial（跨屏拨号）** 是一套电销辅助拨号系统，支持通过电脑控制手机拨号，适用于电销场景。

**核心功能：**
- 电脑端一键拨号，手机实际执行通话
- 支持局域网直连和云中转双通道
- 智能拨号模式（7种）+ 双卡管理
- 通话记录统计、短信辅助发送

## 三、各端功能详解

### 3.1 Android 端（Kotlin）

#### 核心文件与功能

| 文件 | 功能 |
|------|------|
| `MainActivity.kt` | 三 Tab 主界面（ViewPager2），权限管理，主题应用 |
| `ConnectFragment.kt` | 连接管理：UDP 发现、云服务器管理、拨号设置、主题设置 |
| `DialService.kt` | 后台服务：WebSocket 连接、拨号执行、短信、挂断、保活 |
| `CallLogFragment.kt` | 通话记录查询、按天统计、拨号模式快速切换 |
| `DialMode.kt` | 7 种拨号模式枚举 |
| `DialAnimationOverlay.kt` | 拨号动画（烟花、弹跳、结合） |
| `SimSelectOverlay.kt` | 悬浮选卡窗（30秒超时） |
| `SmsConfirmActivity.kt` | 短信确认界面 |
| `ThemeManager.kt` | 主题引擎（16 主题 × 7 亮度） |
| `CallLogDb.kt` | SQLite 数据库（dial_log 表、sim_cache 表） |

#### 拨号模式（7 种）

1. **POPUP（弹窗）**：每次拨号弹出选卡界面
2. **ROUND_SELECT（轮选）**：未识别→循环，已识别→弹窗
3. **OPPOSITE（相反）**：未识别→循环，已识别→用另一张卡
4. **SIM1（卡1）**：始终使用卡1
5. **SIM2（卡2）**：始终使用卡2
6. **ALTERNATE（循环）**：全局交替（卡1→卡2→卡1...）
7. **REMEMBER（记忆）**：记住每个号码上次使用的卡

#### 连接管理

- **局域网发现**：UDP 广播（端口 35433），输入 4 位 PIN 后自动扫描
- **云中转**：支持多服务器管理（添加/删除/测试/排序），自动遍历连接
- **WebSocket**：`ws://ip:35432`（局域网）或 `ws://server:port`（云端）

### 3.2 PC 端（Electron）

#### 核心功能

| 功能 | 说明 |
|------|------|
| 主窗口 | 420×780，无边框，透明背景，可拖动 |
| 悬浮条 | 440×48，始终置顶，可拖动/缩放，显示拨号状态 |
| HTTP 服务 | 端口 35432，提供 `/dial`、`/open`、`/hangup` 等接口 |
| UDP 发现 | 端口 35433，接收手机发现请求，定期广播 |
| 多手机管理 | 支持同时连接多个手机，可切换活跃设备 |
| 云中转 | 连接云中继服务器，支持多服务器遍历 |
| 主题系统 | 与 Android 端一致的 16 套主题 |
| 开机自启 | Windows 注册表实现（无需管理员权限） |

#### HTTP API（端口 35432）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/dial?number=xxx` | GET | 拨号 |
| `/open` | GET | 打开主窗口 |
| `/toggle-floatbar` | GET | 切换悬浮条 |
| `/sms?number=xxx` | GET | 打开短信窗口 |
| `/hangup` | GET | 挂断电话 |
| `/status` | GET | 查询连接状态 |

### 3.3 云中继服务器

#### Python 版（主力，`cloud_relay.py`）

- **协议**：WebSocket（端口 35430，可自定义）
- **功能**：
  - PIN 分组转发（多用户隔离）
  - 手机↔PC 双向消息转发
  - 心跳检测（60 秒超时）
  - HTTP 健康检查（`GET /health`）
  - Web 控制台（`web_server.py` + `dashboard.html`）
- **打包**：PyInstaller 打包为 `AutoDial-Cloud-Relay.exe`
- **日志**：`%APPDATA%/autodial-cloud-relay/cloud-relay.log`

#### Node.js 版（备用，`server.js`）

- 功能与 Python 版类似，作为备用实现

## 四、通信架构

```
[Chrome 插件] --HTTP--> [PC 端] <--WebSocket--> [云服务器] <--WebSocket--> [Android 端]
                        |                          ^
                        |                          |
                        +-------- LAN 直连 ----------+
                                   (优先)
```

**双通道策略：**
1. 优先使用局域网 WebSocket 直连（延迟低）
2. 局域网断开后自动切换到云中转
3. 局域网恢复后自动切回

## 五、端口与协议

| 用途 | 端口 | 协议 |
|------|------|------|
| PC 端 HTTP API | 35432 | HTTP REST |
| PC 端 WebSocket | 35432 | WebSocket |
| PC 端 UDP 发现 | 35433 | UDP 广播 |
| 云中继服务器 | 35430（默认） | WebSocket + HTTP |

### WebSocket 消息协议

**握手：**
- PC → 服务器：`{"type":"pc_hello","pin":"6625","hostname":"xxx"}`
- 服务器 → PC：`{"type":"pc_auth_ok","pin":"6625","phoneCount":0}`
- 手机 → 服务器：`{"type":"phone_hello","pin":"6625","deviceName":"xxx"}`
- 服务器 → 手机：`{"type":"auth_ok","pin":"6625","pcCount":1}`

**业务消息：**
- `dial`：拨号指令 `{"type":"dial","pin":"xxx","number":"13800138000"}`
- `dial_result`：拨号结果 `{"type":"dial_result","number":"xxx","status":"success"}`
- `hangup`：挂断指令
- `sms`：短信指令
- `sms_result`：短信发送结果
- `ping/pong`：心跳（不记日志）

## 六、构建方法

### 6.1 Android 端 APK

**GitHub Actions 自动构建：**
1. 推送代码到 `master` 分支
2. 自动触发 `.github/workflows/build.yml`
3. 构建完成后在 Actions 页面下载 APK

**本地构建：**
```bash
cd AutoDial/android
./gradlew assembleDebug        # 调试版
./gradlew assembleRelease      # 正式版（需要签名配置）
```

**APK 签名：**
- 格式：PKCS12（.p12）
- alias：`autodial`
- 密码：`AutoDial2026!`
- GitHub Secrets：`KEYSTORE_BASE64`、`KEYSTORE_PASSWORD`、`KEY_ALIAS`、`KEY_PASSWORD`

### 6.2 PC 端 EXE

```bash
cd AutoDial/pc-app
npm install
npm run build          # 或 node pack.js
# 输出：AutoDial/pc-app/output/AutoDial-win32-x64/AutoDial.exe
```

### 6.3 云中继服务器 EXE

```bash
cd AutoDial/cloud-relay/python
pip install -r requirements.txt
# 使用 PyInstaller 打包（参考 Launcher.cs 或 launcher.cpp）
# 输出：AutoDial-Cloud-Relay.exe
```

### 6.4 Chrome 插件

1. 打开 Chrome 浏览器
2. 进入 `chrome://extensions/`
3. 开启"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择 `AutoDial-Extension/` 目录

## 八、已知问题与限制

1. **插件浮窗右键菜单**：外部点击有时不关闭（已知 bug）
2. **Android 拨号权限**：需要系统拨号权限，部分 ROM 需要手动授权
3. **挂断功能**：需要 `ANSWER_PHONE_CALLS` 权限（Android 8.0+）
4. **悬浮窗权限**：需要手动在系统设置中开启
5. **电池优化**：需要手动关闭电池优化，否则服务可能被杀
